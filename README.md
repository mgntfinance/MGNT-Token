## ⚡ Magnetic Polarity Revenue Routing

The core innovation of Magnet Finance. Revenue is not pooled and distributed generically — it is routed by *character* to match the token that earned it.

### Three Revenue Streams

| Stream | Source | Mechanism |
|--------|--------|-----------|
| Swap Fees | Magnetic Swap (Jupiter v6) | 30bps platform fee to treasury |
| LP Yield | Meteora DLMM concentrated liquidity | mgntSOL/SOL pool |
| Treasury Yield | Diversified yield positions | Funded by presale proceeds |

### Two-Tier Distribution — Weekly Magnetic Epoch

Every 7 days, all protocol revenue is tallied and distributed on-chain.

**Tier 1 — Base Layer (every holder):**
A fixed percentage of total protocol revenue distributes proportionally to ALL $MGNT, pMGNT, and nMGNT holders every epoch. No token holder earns zero.

**Tier 2 — Polarity Amplifiers (by token type):**

| Token | Amplifier Source | Logic |
|-------|-----------------|-------|
| $MGNT | Swap fee revenue | Core utility token earns from core protocol activity |
| pMGNT | LP yield | Positive polarity / volume token earns from volume-driven yield |
| nMGNT | Treasury yield | Negative polarity / scarcity token earns from long-duration compounding |

### Staking is Always Double-Up, Never a Sacrifice

Staking $MGNT to receive pMGNT or nMGNT is always additive. Holders never lose their $MGNT earnings position by staking.

- **Unstaked $MGNT earns:** Base layer + swap fee amplifier
- **Staked $MGNT (→ pMGNT or nMGNT) earns:** Base layer + swap fee amplifier + polarity amplifier

Lock duration multiplies amplifier weight. The longer you lock, the more you earn.
